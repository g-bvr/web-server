WebserverPlugin

Purpose

operates a web server to handle http requests and trigger serialized script execution

Commands

CommandDescription
WebserverInitCommandStart web server, listening to specified port
WebserverEndPointCommandDefine an endpoint (endpoints are executed concurrently, the request returns after processing is done, endpoints can have url parameters that are passed as variables to the script)
WebserverTriggerCommandDefine a trigger (triggers are queued and executed sequentially, the request returns directly after the trigger script has been queued, triggers cannot have url parameters)
WebserverShutdownCommandShutdown webserver (no requests/triggers are accepted any longer, server shuts down after all scheduled trigger runs have terminated)
